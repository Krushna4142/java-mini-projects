## Why this project?

This project builds confidence with:

- Taking user input
- Writing clean methods
- Applying basic control flow

These are the first things interviewers expect a Java developer to be comfortable with.

---

## Key Learnings

- How to use `Scanner` for input
- Why methods improve readability
- How `switch-case` simplifies decision making
- Importance of separating logic into functions

---

## Interview Angle

An interviewer may ask:

- Why did you use methods instead of writing logic in `main`?
- How would you handle invalid input?
- How would you extend this calculator?

This project gives clear answers to all of the above.

---

## Best Practice Followed

- Meaningful class name
- Single responsibility per method
- Clean console output
